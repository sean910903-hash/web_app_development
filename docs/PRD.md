# PRD - 任務管理系統

## 1. 專案概述
- **背景與動機**：在日常生活或學習中，常常會有很多待辦事項需要記錄與追蹤，例如作業、報告、會議或生活瑣事。如果沒有一個清楚的管理工具，很容易忘記或延誤重要任務，因此需要一個簡單實用的任務管理系統來協助整理工作。
- **目標用戶**：我自己（個人使用的任務管理工具）。
- **核心價值主張**：提供一個清楚、簡單、好操作的系統，讓使用者可以新增、查看、編輯、刪除任務，並掌握每個任務的完成狀態，提升做事效率。
## 2. 功能需求
- **新增任務**
   - 作為我自己，我希望可以新增任務，包含任務名稱、內容說明、截止日期與完成狀態，以方便我記錄待辦事項。
- **查看任務**
   - 作為我自己，我希望可以查看目前所有任務的列表，方便我快速掌握有哪些事情要做。
- **編輯任務**
   - 作為我自己，我希望可以修改任務內容，例如名稱、說明或截止日期，讓任務資訊保持正確。
- **刪除任務**
   - 作為我自己，我希望可以刪除不再需要的任務，避免列表過於雜亂。
- **標記完成狀態**
   - 作為我自己，我希望可以將任務標記為已完成或未完成，以便追蹤任務進度。
- **篩選任務**
   - 作為我自己，我希望可以篩選已完成與未完成任務，方便我快速查看目前尚未完成的事項。

## 3. 非功能需求
- **技術限制**：後端採用 Flask 框架，前端使用 Jinja2 模板引擎進行渲染，資料庫使用輕量的 SQLite。
- **效能與安全考量**：
   - 介面操作應簡單直觀，提供良好的單人使用體驗。
   - 實作基本的輸入驗證，避免不正確或空白資料寫入系統。
   - 需能正確處理新增、編輯、刪除與狀態切換等操作。
   - 由於設定為個人單人使用系統，暫時不需要登入驗證或高併發負載處理。

## 4. MVP 範圍
- **Must Have（必要功能）**：
 - 新增任務（包含任務名稱、內容說明、截止日期）。
  - 瀏覽任務列表與詳細內容。
  - 透過關鍵字搜尋任務。
  - 編輯與刪除任務。
  - 標記完成狀態。
  - 篩選任務。
- **Should Have（重要功能）**：
  - 依截止日期排序任務。
  - 任務列表顯示完成與未完成數量。
- **Nice to Have（次要功能）**：
  - 任務分類標籤。
  - 任務優先順序功能。
  - 任務提醒功能。
